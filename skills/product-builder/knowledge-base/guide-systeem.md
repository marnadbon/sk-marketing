# Guide Systeem — Van Lead Magnet tot Standalone Product

> Ontwerp guides die de lezer echt verder helpen: van een compacte lead magnet die e-mailadressen genereert, tot een authority guide die je als dé expert in je niche positioneert. Met werkbladen, templates, en een heldere CTA.

---

## Guide Typen

### Lead Magnet Guide (5-15 pagina's)

| Aspect | Specificatie |
|--------|-------------|
| **Doel** | E-mailadressen verzamelen |
| **Prijs** | Gratis |
| **Volume** | Hoog — veel downloads |
| **Scope** | Specifiek probleem, snelle oplossing |
| **Leestijd** | 15-30 minuten |
| **Format** | "De 5 stappen om..." / "Checklist voor..." / "Template: ..." |

**Kwaliteitseis:** zo goed dat mensen verbaasd zijn dat het gratis is. Een slechte lead magnet = slechte eerste indruk.

**Leidt naar:** webinar, cursus, of gesprek (de CTA is cruciaal)

### Standalone Guide (30-80 pagina's)

| Aspect | Specificatie |
|--------|-------------|
| **Doel** | Omzet + autoriteit |
| **Prijs** | €17-97 |
| **Volume** | Gemiddeld |
| **Scope** | Compleet systeem of methode |
| **Leestijd** | 1-3 uur |
| **Format** | Vervangt een mini-cursus voor lezers die liever lezen |

Bevat werkbladen en templates. Is zelfstandig bruikbaar — de lezer kan de transformatie bereiken zonder iets anders te kopen.

**Leidt naar:** signature cursus of coaching (als die bestaan)

### Authority Guide (60-150 pagina's)

| Aspect | Specificatie |
|--------|-------------|
| **Doel** | Positionering als dé expert |
| **Prijs** | Gratis of betaald |
| **Volume** | Laag-gemiddeld |
| **Scope** | Book-level diepte in guide format |
| **Leestijd** | 3-8 uur |
| **Format** | Origineel framework, data, case studies |

Wordt gebruikt in marketing: "Schrijver van [guide naam]." Bevat origineel denken, niet alleen samenvatting van bestaande kennis.

**Leidt naar:** speaking, consulting, high-ticket programma's

### Companion Guide (10-30 pagina's)

| Aspect | Specificatie |
|--------|-------------|
| **Doel** | Cursus of workshop verrijken |
| **Prijs** | Onderdeel van ander product |
| **Volume** | Gekoppeld aan hoofdproduct |
| **Scope** | Samenvattingen, werkbladen, quick references |
| **Format** | Niet standalone — altijd bijlage |

Verhoogt de perceived value van het hoofdproduct. Deelnemers hebben iets "tastbaars" naast video's.

## Structuur per Type

### Lead Magnet Structuur

```
Cover + Titel (pakkend, specifiek resultaat)
Intro (1 pagina): probleem + belofte + voor wie
Kern (3-10 pagina's): de oplossing in 3-7 stappen
Samenvatting (1 pagina): key takeaways
CTA (1 pagina): volgende stap (webinar, cursus, gesprek)
```

**Totaal: 5-15 pagina's.** Niet langer — een lead magnet die niet gelezen wordt, werkt niet.

### Standalone Guide Structuur

```
Cover + Titel + Ondertitel
Inhoudsopgave
Introductie (2-3 pagina's): probleem, belofte, voor wie, hoe te gebruiken
Hoofdstuk 1-6 (elk 4-10 pagina's): kerncontent + oefeningen
Samenvatting & Actieplan (2-3 pagina's)
Bonushoofdstuk (optioneel): advanced tips of FAQ
Werkbladen & Templates (appendix)
Over de Auteur + CTA
```

**Totaal: 30-80 pagina's.** Genoeg voor een complete transformatie, niet zo lang dat niemand het afmaakt.

### Authority Guide Structuur

<!-- BRON: authority positioning via geschreven content -->

```
Cover + Ondertitel
Voorwoord (optioneel: door expert in het veld)
Inhoudsopgave
Introductie: het probleem, waarom dit boek, de these
Deel I: Context & Achtergrond (2-3 hoofdstukken)
Deel II: Het Framework (3-5 hoofdstukken — de kern)
Deel III: Implementatie (2-3 hoofdstukken — toepassing)
Conclusie: samenvatting + visie
Appendix: data, bronnen, werkbladen
Over de Auteur + CTA
```

**Totaal: 60-150 pagina's.** Dit is in feite een boek — behandel het ook zo qua kwaliteit.

### Companion Guide Structuur

Afgeleid van het hoofdproduct:

```
Cover + "Behorend bij [Cursus/Workshop naam]"
Overzicht: hoe deze guide te gebruiken
Per module/blok: samenvatting + werkblad
Quick reference: de belangrijkste frameworks/modellen
Checklist: alle actiepunten op een rij
```

## Hoofdstuk Design Pattern

<!-- BRON: technisch schrijven en instructional design patterns -->

### Kort Hoofdstuk (Lead Magnet, 1-3 pagina's)

```
1. Kernvraag — één zin die het hoofdstuk samenvat als vraag
2. Antwoord — directe oplossing (geen omwegen)
3. Voorbeeld — concreet (1 alinea)
4. Actie — doe dit nu (1-2 zinnen)
```

**Principe:** Snel, direct, waardevol. De lezer moet na 2 minuten iets bruikbaars hebben.

### Standaard Hoofdstuk (Standalone, 4-8 pagina's)

```
1. Opening — waarom dit onderwerp ertoe doet (motivatie)
2. Context — achtergrond, definities, "waar staan we"
3. Kerninhoud — de methode/kennis (met subkoppen voor overzichtelijkheid)
4. Voorbeeld of case study — concreet bewijs
5. Veelgemaakte fouten — wat je NIET moet doen
6. Werkblad of oefening — directe toepassing
7. Key takeaways — 3-5 bullets
```

**Principe:** Compleet maar toegankelijk. Elk hoofdstuk kan bijna standalone gelezen worden.

### Diep Hoofdstuk (Authority, 8-15 pagina's)

```
1. Openingsverhaal of stelling — haak de lezer (contrarian, verrassend, herkenbaar)
2. Het probleem — uitgebreide context (waarom bestaande oplossingen tekortschieten)
3. Het framework/de theorie — originele bijdrage (het hart van het hoofdstuk)
4. Bewijs — data, case studies, voorbeelden (minimaal 2-3)
5. Implementatie — hoe toe te passen (stap voor stap)
6. Edge cases & nuance — wanneer werkt dit NIET (eerlijkheid = geloofwaardigheid)
7. Werkblad — toepassing op de situatie van de lezer
8. Samenvatting + overgang naar volgend hoofdstuk
```

**Principe:** Diep, origineel, overtuigend. De lezer moet denken "deze persoon begrijpt dit écht."

## Werkbladen & Templates

### Werkblad Ontwerp Regels

1. **Eén doel per werkblad** — niet te veel vragen tegelijk
2. **Voorbeeld ingevuld** — laat zien hoe het eruitziet als het af is
3. **Duidelijke instructies** — stap-voor-stap wat de lezer moet doen
4. **Genoeg ruimte** — bij PDF: invulvelden groot genoeg voor echte antwoorden
5. **Standalone bruikbaar** — werkblad moet ook zonder het hoofdstuk te begrijpen zijn
6. **Visueel helder** — kaders, labels, nummering

### Template Typen

| Type | Beschrijving | Voorbeeld |
|------|-------------|-----------|
| **Invultemplate** | Vul de blanks in | Elevator pitch template: "Ik help [wie] met [wat] zodat [resultaat]" |
| **Checklist** | Vink af wat je gedaan hebt | "SEO-checklist voor je blogpost" |
| **Scorecard** | Beoordeel jezelf op criteria | "Hoe sterk is je productidee? Score 1-5 op 8 criteria" |
| **Planner** | Vul je plan in | "90-dagen lanceringplan" |
| **Swipe file** | Voorbeelden om te kopiëren en aan te passen | "10 e-mail onderwerpsregels die werken" |

### Werkblad Minimum per Guide Type

| Guide type | Minimum werkbladen |
|-----------|-------------------|
| Lead magnet | 1 (bijv. de kerntemplate) |
| Standalone | 1 per 2 hoofdstukken |
| Authority | 1 per 2-3 hoofdstukken + appendix |
| Companion | 1 per module/blok van het hoofdproduct |

## Van Guide naar Cursus Pipeline

Een guide kan de basis zijn voor een cursus:

| Guide Element | Wordt in Cursus |
|---------------|-----------------|
| Hoofdstuk | Module |
| Subsectie | Les |
| Werkblad | Oefening |
| Voorbeeld | Case Study Les |
| Key Takeaways | Module Samenvatting |

### Uitbreidingsstrategie

<!-- BRON: product ladder strategie, repurposing methodologie -->

1. **Elk hoofdstuk → module** — voeg video-uitleg toe
2. **Elk werkblad → interactieve oefening** — met feedback loop
3. **Voeg community toe** — plek om vragen te stellen
4. **Voeg Q&A/coaching toe** — live sessies
5. **Prijs: guide × 3-10 = cursus prijs**

### Waarom Dit Werkt

- De guide bewijst dat het content waardevol is (validatie)
- Lezers die de guide kochten, zijn warme leads voor de cursus
- De structuur is al bewezen — je hoeft alleen te verdiepen
- Testimonials van guide-lezers = social proof voor de cursus

## Introductie Schrijfwijzer

**Schrijf de introductie als LAATSTE** — je weet dan precies wat erin staat.

### Lead Magnet Introductie (1 pagina)

```
Het probleem (2-3 zinnen): [herkenbaar probleem]
De belofte (1 zin): "In deze guide leer je [concreet resultaat]"
Voor wie (1-2 zinnen): "Deze guide is voor [specifieke persoon]"
```

### Standalone/Authority Introductie (2-3 pagina's)

```
Voor wie is deze guide: [specifieke beschrijving]
Wat je gaat leren: [transformatiebelofte — concreet]
Hoe je deze guide gebruikt: [leeswijzer]
Over de auteur: [korte bio — 2-3 zinnen, focus op geloofwaardigheid]
```

## Guide Quality Checklist

### Type & Scope
- [ ] Guide type is bewust gekozen (lead magnet/standalone/authority/companion)
- [ ] Lengte past bij het type
- [ ] Scope is realistisch voor het gekozen formaat

### Inhoud
- [ ] Transformatiebelofte is helder in de intro
- [ ] Elk hoofdstuk volgt het juiste design pattern
- [ ] Voorbeeld(en) in elk hoofdstuk
- [ ] Key takeaways aan het einde van elk hoofdstuk (standaard + diep)

### Werkbladen
- [ ] Minstens 1 werkblad of template per 2 hoofdstukken
- [ ] Alle werkbladen hebben een ingevuld voorbeeld
- [ ] Werkbladen zijn standalone bruikbaar

### Structuur
- [ ] Inhoudsopgave klopt met de werkelijke structuur
- [ ] Logische volgorde (bouwt op, geen sprongen)
- [ ] Introductie geschreven als laatste (klopt met inhoud)

### CTA
- [ ] CTA is helder en past bij het guide type
- [ ] Lead magnet → duidelijke volgende stap (webinar/cursus/gesprek)
- [ ] Standalone → optionele upsell naar cursus/coaching
- [ ] Authority → positionering + volgende stap
- [ ] Companion → terug naar het hoofdproduct
