# Cursus Architectuur — Curriculum Design & Structuur

> Hoe je een cursus ontwerpt die daadwerkelijk leert: van transformatiebelofte tot uitgewerkt curriculum, met de juiste module- en lespatronen, progressie, scaffolding, en oefeningen.

---

## De Transformatiebelofte als Fundament

<!-- BRON: curriculum design methodologie, backwards design -->

Elke cursus begint met het einde: **wat kan de leerling DOEN na het voltooien?**

### Achteruit Werken
1. Definieer de transformatie: "Na deze cursus kan de leerling [concreet resultaat]"
2. Welke vaardigheden/kennis zijn nodig voor dat resultaat?
3. In welke volgorde moeten die geleerd worden?
4. Wat is de kleinste stap waarmee je kunt beginnen?

### De Belofte Bepaalt de Scope
- Alles in de cursus dient de belofte — niets anders
- "Leuk om te weten" ≠ "nodig voor de transformatie"
- Te breed = de leerling bereikt niets. Te smal = niet genoeg waarde.
- **Test:** als je een les kunt verwijderen zonder dat de transformatie in gevaar komt, was die les niet nodig.

## Snelle Sizing Formule

### De 4×5×5 Vuistregel
Een standaard cursus = **4 secties × 5 video's × 5 minuten = 1,5 uur totaal.**

| Cursusniveau | Secties | Video's/sectie | Min/video | Totaal video's | Totale duur |
|---|---|---|---|---|---|
| Mini-cursus | 3-4 | 4-5 | 3-5 min | 12-20 | 45-90 min |
| Signature | 4-6 | 5-7 | 5-10 min | 20-40 | 2-5 uur |
| Premium | 6-8 | 5-8 | 5-15 min | 30-60 | 4-10 uur |

### Waarom dit helpt
- Voorkomt "scope creep" — je weet van tevoren hoeveel je maakt
- Maakt planning concreet — "ik moet 20 video's van 5 minuten filmen"
- Verlaagt de drempel — 20 korte video's klinkt haalbaarder dan "een hele cursus"
- Niet letterlijk nemen — sommige secties hebben meer of minder video's

## Structuur Niveaus

```
Cursus
  └── Module (groot thema, 3-8 per cursus)
       └── Les (specifiek onderwerp, 3-10 per module)
            └── Onderdelen (video, tekst, oefening, quiz, werkblad)
```

### Module = Groot Thema
- Een module behandelt één aspect van de transformatie
- Elke module heeft een eigen sub-transformatie: "Na deze module kan de leerling..."
- 3-8 modules per cursus (afhankelijk van pricing tier)

### Les = Specifiek Onderwerp
- Een les behandelt één leerpunt
- 3-10 lessen per module
- Elke les heeft: leerdoel → kerninhoud → voorbeeld → actie
- Ideale lengte: 5-15 minuten (video) of 500-1500 woorden (tekst)

### Onderdeel = Lescomponent
- Video, tekst, oefening, quiz, werkblad, checklist
- Mix van formats houdt aandacht vast
- Niet elk onderdeel is verplicht, maar elke les heeft minimaal: inhoud + actie

## Module Patronen (4 Typen)

<!-- BRON: instructional design patterns, curriculum structuur modellen -->

### Patroon 1 — Lineair (Stap-voor-stap)

```
Module 1 → Module 2 → Module 3 → Module 4
```

- Elke module bouwt voort op de vorige
- Strenge volgorde — overslaan is niet mogelijk
- **Best voor:** skill-building, processen, methodologieën
- **Voorbeeld:** SEO cursus (research → on-page → content → linking)

### Patroon 2 — Hub & Spoke

```
        Module 2
          ↑
Module 1 → Hub ← Module 3
          ↓
        Module 4
```

- Centraal concept met branches
- Modules kunnen in willekeurige volgorde na de intro
- **Best voor:** toolkits, referentie-cursussen, meerdere kanalen
- **Voorbeeld:** "Social Media Mastery" (kernstrategie → LinkedIn / Instagram / YouTube)

### Patroon 3 — Problem-Solution

```
[Probleem 1 → Oplossing 1] [Probleem 2 → Oplossing 2] [Probleem 3 → Oplossing 3]
```

- Elke module adresseert één probleem
- Losjes sequentieel maar kan standalone staan
- **Best voor:** troubleshooting, veelvoorkomende uitdagingen
- **Voorbeeld:** "Fix Your Marketing" (traffic → conversie → retentie → schaling)

### Patroon 4 — Chronologisch

```
Dag 1 → Dag 7 → Dag 14 → Dag 21 → Dag 30
```

- Volgt een tijdlijn
- **Best voor:** implementatieprogramma's, challenges
- **Voorbeeld:** "30-Dagen LinkedIn Challenge"

### Patroon Kiezen

| Situatie | Patroon |
|----------|---------|
| Alles bouwt op alles voort | Lineair |
| Centraal concept met vertakkingen | Hub & Spoke |
| Onafhankelijke problemen | Problem-Solution |
| Tijdgebonden implementatie | Chronologisch |

## Les Patronen (4 Typen)

### Patroon A — Concept Uitleg

Gebruik voor: theoretische lessen, nieuwe begrippen, mindset shifts

1. **Waarom dit belangrijk is** (motivatie — de leerling moet WILLEN leren)
2. **Wat het is** (definitie + context — helder en jargon-vrij)
3. **Hoe het werkt** (mechanisme — het systeem achter het concept)
4. **Voorbeeld** (concreet — "in de praktijk ziet dit er zo uit")
5. **Actie:** pas dit toe op jouw situatie

### Patroon B — Vaardigheid Aanleren

Gebruik voor: praktische skills, tools, technieken

1. **Wat je gaat leren** (doel — concreet resultaat)
2. **Demonstratie** (laat het zien — screen share, walkthrough, video)
3. **Stap-voor-stap breakdown** (elk detail uitgelegd)
4. **Veelgemaakte fouten** (wat beginners typisch fout doen)
5. **Actie:** oefen deze vaardigheid (specifieke opdracht)

### Patroon C — Framework Introductie

Gebruik voor: modellen, systemen, methodes

1. **Het probleem dat dit framework oplost** (herkenning)
2. **Overzicht van het framework** (big picture — alle elementen)
3. **Elk element uitgelegd** (detail per onderdeel)
4. **Case study** (het framework in actie — bewijs dat het werkt)
5. **Actie:** vul het framework in voor jouw situatie (werkblad)

### Aantrekkelijke Lestitels — Verkoop Bevestigen Na De Koop

#### Het Principe
De meeste makers stoppen met moeite doen ná de verkoop. Maar elke lestitel is een kans om de koper te bevestigen: "Je hebt een goede investering gedaan."

#### Van functioneel naar aantrekkelijk
| Saai (functioneel) | Aantrekkelijk (verkoopbevestigend) |
|---|---|
| "Kies je onderwerp" | "Hoe je een cursus-idee kiest dat iedereen wil kopen" |
| "Maak afbeeldingen voor je product" | "Afbeeldingen die iedereen je product doen willen" |
| "Ga door je kledingkast" | "Je hebt al minstens 50 Pinterest-outfits in je kast" |

#### Regels
1. Lestitels herschrijven als LAATSTE stap — nadat de inhoud af is
2. Titels moeten de belofte van de les benadrukken, niet de taak
3. Gebruik de taal van het resultaat, niet de taal van het proces
4. De koper moet bij het zien van de inhoudsopgave denken: "Wauw, dit is precies wat ik nodig heb"

#### Waarom dit ertoe doet
- Versterkt de aankoop-beslissing (minder refund requests)
- Verhoogt de completion rate (titels trekken de leerling naar de volgende les)
- Creëert goodwill → reviews → herhaalaankopen

---

### Patroon D — Case Study / Voorbeeld

Gebruik voor: bewijs, inspiratie, toepassing

1. **Context** (de situatie — wie, wat, waar)
2. **Uitdaging** (wat was het probleem?)
3. **Aanpak** (wat werd er gedaan? — link naar eerder geleerde methode)
4. **Resultaat** (wat gebeurde er? — concreet met cijfers)
5. **Actie:** pas de lessen toe op jouw situatie

## Progressie & Scaffolding

<!-- BRON: scaffolding theorie, zone of proximal development -->

### Scaffolding Principes

- **Elke les bouwt voort op de vorige** — ga nooit uit van kennis die niet is geleerd
- **Start met de simpelste versie**, voeg complexiteit toe
- **Quick wins eerst** — vroeg succes bouwt momentum en vertrouwen
- **Moeilijkheidscurve:** zacht begin → steiler midden → plateau aan het eind

### Progressie Ontwerp

| Positie | Module | Functie |
|---------|--------|---------|
| **Module 1** | Onboarding | Verwachtingen, tools, mindset, eerste quick win |
| **Module 2** | Eerste quick win | Snel resultaat voor motivatie — de leerling ziet dat het werkt |
| **Middenmodules** | Kernstof | Hier zit de diepte — de echte transformatie |
| **Voorlaatste** | Advanced | Moeilijkste stof — voor wie alles beheerst |
| **Laatste** | Integratie | Alles samenvoegen, next steps, diploma/afronding |

### Doorverwijzingen (Binnen Cursus)

Gebruik doorverwijzingen om het grotere plaatje te laten zien:
- **Preview:** "In Module 3 leer je hoe je dit toepast op..."
- **Terugverwijzing:** "Zoals je in Les 2.1 geleerd hebt..."
- **Motivatie:** "Dit lijkt nu veel, maar in Module 5 valt alles op zijn plek"

## Minder Is Meer — De Content Filter

Beginners denken: meer modules, meer lessen, meer waarde. Professionals weten: minder content = hogere completion rate = betere resultaten = meer mond-tot-mond reclame.

### De Filterregel
Voor ELKE les die je wilt toevoegen, stel deze vraag:
"Is deze les ESSENTIEEL om het eindresultaat te bereiken?"
- Ja → behouden
- Nee → schrappen. Hoe goed de les ook is.

### Waarom dit werkt
- Completion rates: <20% van cursuskopers maakt de cursus af
- Elke extra les verlaagt dat percentage
- Een kortere cursus die VOLLEDIG wordt afgemaakt levert meer resultaat dan een lange cursus waarvan niemand voorbij module 3 komt
- Resultaat = testimonials = social proof = meer verkoop

### De "Voegt dit toe aan de transformatie?" test
Niet: "Is dit interessante informatie?"
Maar: "Kan de leerling de transformatie bereiken ZONDER dit?"
Als ja → het is nice-to-have, geen must-have. Schrap het of maak er een bonus van.

## Oefeningen & Werkbladen

### Oefening Typen

| Type | Beschrijving | Wanneer | Geschatte tijd |
|------|-------------|---------|----------------|
| **Invuloefening** | Vul een template in met eigen info | Na framework introductie | 10-20 min |
| **Mini-opdracht** | Doe iets kleins en concreets | Na elke les | 5-15 min |
| **Project** | Grotere opdracht die meerdere lessen combineert | Einde module | 30-60 min |
| **Reflectie** | Schrijf op wat je geleerd hebt en hoe je het gaat toepassen | Einde module | 10 min |
| **Peer review** | Deel je werk en geef feedback | Alleen bij community-cursussen | 20-30 min |

### Werkblad Ontwerp Regels

1. **Eén doel per werkblad** — niet te veel vragen tegelijk
2. **Voorbeeld ingevuld** — laat zien hoe het eruitziet als het af is
3. **Duidelijke instructies** — stap-voor-stap bovenaan
4. **Genoeg ruimte** — invulvelden groot genoeg
5. **Standalone bruikbaar** — het werkblad moet ook zonder de les te begrijpen zijn
6. **Gekoppeld aan specifieke les** — verwijs expliciet naar de les

### Oefening per Module — Minimum

| Cursusniveau | Minimum oefeningen per module |
|--------------|------------------------------|
| Mini-cursus | 1 mini-opdracht + 1 invuloefening |
| Signature | 2-3 mini-opdrachten + 1 project + 1 werkblad |
| Premium | 3+ mini-opdrachten + 1 project + 1 werkblad + reflectie |

## Onboarding Module (Module 0 of 1)

<!-- BRON: onboarding best practices voor online cursussen -->

Elke cursus heeft een onboarding module. Dit is de meest onderschatte module:

### Onboarding Structuur

1. **Welkom** — wie is de cursusmaker, wat is de belofte, waarom deze cursus
2. **Wat je kunt verwachten** — structuur, tijdsinvestering, werkwijze
3. **Technische setup** — tools, accounts, installatie (als nodig)
4. **Community richtlijnen** — als er een community is: gedragsregels, hoe je hulp vraagt
5. **Eerste quick win** — een simpele actie die direct resultaat geeft

### Emotionele Opening (Les 1: Welkom)
- "Gefeliciteerd met je investering in jezelf" — bevestig hun keuze
- "Ik ben trots op je" — de gemiddelde persoon hoort dit te weinig
- Leg uit wat ze gaan bereiken (belofte herhalen)
- Stel gerust: je hoeft het niet perfect te doen, begin gewoon

### Waarom Onboarding Cruciaal Is

- **Commitment:** de leerling heeft net betaald — bevestig dat het de juiste keuze was
- **Verwachtingen:** voorkom teleurstelling door helder te zijn over wat wel/niet in de cursus zit
- **Momentum:** een vroege quick win zorgt dat de leerling doorgaat
- **Drop-off:** de meeste leerlingen stoppen in de eerste week — een goede onboarding voorkomt dit

## Afsluitingsmodule (Laatste Module)

Spiegelt de opening — sluit de emotionele cirkel:

### Structuur
1. **Samenvatting** — wat heb je allemaal geleerd? (module voor module, 1 zin)
2. **Resultaat benoemen** — "Je hebt nu [transformatie]. Dat is enorm."
3. **Trots uitspreken** — "Ik ben trots op je. De meeste mensen beginnen niet eens."
4. **Volgende stappen** — wat kan de leerling NU doen met deze kennis?
5. **Community** — blijf verbonden (als er een community is)
6. **CTA** — volgende product, review achterlaten, of resultaat delen

### Waarom een sterke afsluiting cruciaal is
- De "recency effect": het laatste wat iemand ervaart bepaalt hoe ze het hele product beoordelen
- Een sterke afsluiting = betere reviews, meer mond-tot-mond, hogere tevredenheid
- Emotioneel: de leerling voelt zich gezien en gewaardeerd

## Curriculum Quality Checklist

### Structuur
- [ ] Transformatiebelofte is helder en meetbaar
- [ ] Module-structuur volgt een logisch patroon (lineair/hub-spoke/problem-solution/chronologisch)
- [ ] Elke module heeft een eigen sub-transformatie
- [ ] Totale scope past bij het prijsniveau

### Lessen
- [ ] Elke les heeft een leerdoel + actie
- [ ] Elk leerdoel is concreet en meetbaar ("na deze les kan de leerling...")
- [ ] Elk lespatroon past bij het type content (concept/vaardigheid/framework/case study)
- [ ] Geen les zonder voorbeeld

### Progressie
- [ ] Scaffolding: moeilijkheid bouwt geleidelijk op
- [ ] Quick win in module 1 of 2
- [ ] Onboarding module aanwezig
- [ ] Alle jargon uitgelegd bij eerste gebruik

### Oefeningen
- [ ] Geen module zonder oefening
- [ ] Mix van oefentypen (invul, mini-opdracht, project)
- [ ] Werkbladen hebben ingevulde voorbeelden
- [ ] Oefeningen passen bij het cursusniveau

### Samenhang
- [ ] Doorverwijzingen (previews + terugverwijzingen)
- [ ] Transities tussen lessen ("nu je X weet, gaan we naar...")
- [ ] Gaps uit de gap analyse zijn ingevuld
- [ ] Één cursus = één transformatie (niet alles tegelijk)
